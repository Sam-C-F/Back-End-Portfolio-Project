Pathways for API

  • /api
  • See /api for a more detailed breakdown of all available endpoints, including acceptable queries, example requests and example responses.
  • /api/topics
  • This endpoint allows two types of request, GET and POST. Get shows a list of all topics (or 'slugs') and includes a description. POST allows a new topic to be added.
  • /api/articles/
  • Shows a list of all articles. Allows a POST request which will add a new article to the database. There are a number of queries that can be made to this URL:
    • topic: searches for all articles with the specified topic.
    • sort_by: sorts the data, alphabetically or numerically, by title, topic, author, comment_count, article_id and, by default, created_at.
    • order: either ASC (ascending) or, by default, DESC (descending) (not case sensitive).
    • limit: limits the number of items on a page, by default this is 10.
    • p: selects which page to view if there are more articles than the selected limit.
  • /api/users
  • Shows a list of all users including name, username and the avatar_URL.
  • /api/articles/:article_id
  • Search for an article by article ID. Also allows a PATCH to update the number of votes associated with an article. You will need to manually add the number in place of the ':article_id'.
  • /api/articles/:article_id/comments
  • This endpoint allows two types of request, GET and POST, either search for all comments on an article by article ID, or post a new comment to a specific article. It allows 2 queries 'limit' and 'p' which limit the number of comments shown and changes the page being viewed. You will need to manually add the number in place of the ':article_id'.
  • /api/comments/:comment_id
  • This endpoint allows a DELETE request and removes the comment associated with the given comment_id. Also allows a PATCH to update the number of votes associated with a comment
  • /api/users/username
  • Search for a user by their username. You will need to manually add the username in place of the ':username'
